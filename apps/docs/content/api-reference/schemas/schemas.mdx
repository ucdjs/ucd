# Schemas

Zod schemas that define the shared data contracts across the UCD.js monorepo — API responses, store manifests, lockfiles, and Unicode metadata.

## Installation

```package-install
@ucdjs/schemas
```

## Overview

The `@ucdjs/schemas` package provides Zod schemas for validating Unicode data files, API responses, and internal configurations with full type safety. All schemas are exported from the package root.

## Schema Families

The source is split into five files, each covering a logical contract area.

### API Schemas

Contracts for HTTP API responses and service discovery.

- `ApiErrorSchema` — Standard error response used by all API endpoints.
- `UCDWellKnownConfigSchema` — Shape of the `.well-known/ucd-config.json` endpoint used by clients to discover API paths.

### File System Schemas

Shapes for file entries and the top-level store manifest.

- `FileEntrySchema` / `FileEntryListSchema` — A file or directory entry returned by the files API, with `name`, `path`, and `lastModified`.
- `UCDStoreManifestSchema` — Store-wide manifest keyed by Unicode version string, listing expected files per version.

### Lockfile & Snapshot Schemas

State shapes used by the `ucd` CLI and the `ucd-store` app.

- `LockfileSchema` — The `ucd.lock` file, including per-version snapshot references, timestamps, and optional path filters.
- `SnapshotSchema` — Per-version snapshot containing file paths, SHA-256 hashes (with and without Unicode header), and sizes.

### Version Manifest Schemas

Per-version metadata served from `/.well-known/ucd-store/{version}.json`.

- `ExpectedFileSchema` — A single expected file with `name`, API `path`, and `storePath` (without `/ucd/` prefix).
- `UCDStoreVersionManifestSchema` — Full per-version manifest listing all expected files for a Unicode version.

### Unicode Schemas

Shapes for Unicode version metadata and file-tree responses.

- `UnicodeVersionSchema` / `UnicodeVersionListSchema` — A Unicode version with its URL, documentation URL, date, type (`draft` / `stable` / `unsupported`), and optional `mappedUcdVersion`.
- `UnicodeVersionDetailsSchema` — Extends `UnicodeVersionSchema` with character, block, and script statistics.
- `UnicodeFileTreeNodeSchema` / `UnicodeFileTreeSchema` — Recursive file-tree structure for a Unicode version directory.

## Usage

```typescript
import { UnicodeVersionSchema, LockfileSchema } from "@ucdjs/schemas";

// Parse a Unicode version object
const version = UnicodeVersionSchema.parse(rawVersion);

// Safely parse a lockfile
const result = LockfileSchema.safeParse(rawLockfile);
if (result.success) {
  console.log(result.data.versions);
}
```
