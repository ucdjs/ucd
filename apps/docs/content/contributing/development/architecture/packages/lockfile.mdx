`@ucdjs/lockfile` owns persistent store state and integrity metadata.

## Role

- Reads and writes `.ucd-store.lock` and per-version snapshots.
- Defines the canonical on-disk state used by mirroring and verification flows.
- Provides the persistence contract shared by storage tooling.

## Notes

Use this page for lockfile semantics, authority, and compatibility decisions.
